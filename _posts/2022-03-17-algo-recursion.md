## **🔁 [재귀란 무엇인가?](https://maz2r.github.io/algorithm/algo-recursion/)**

 ***재귀***란 어떠한 것을 정의할 때, 그것을 재참조하여 정의하는 방법을 뜻한다. 말이 조금 어려운 것 같다. [B.O.J. 1629번](https://maz2r.github.io/p_s_/boj-1629/)을 풀 때 정의했던 함수를 살펴보자.
 ```cpp
 int func(int A, int B, int C){
  if(B == 1){

    return A % C;
  }

  long long val = func(A, B / 2, C);
  
  val = val * val % C;
  if(B % 2 == 0){
    
    return val;
  }
  val = val * A % C;

  return val;
}
 ```
 위 함수는 재귀라는 방법을 이용하여 정의한 함수이다. 즉, 이 함수의 정의에서 이 함수가 재참조된 것이다. 그리고 이렇게 만들어진 함수를 ***재귀함수***라고 부른다.<br><br>

## **💣 수학적 귀납법**

 위와 같은 재귀함수를 알고리즘에 이용하려면 먼저 수학적 귀납법에 대해 이해해야 한다. 수학적 귀납법이란 모든 자연수가 어떠한 성질을 만족한다는 명제를 증명하는 방법이다. 가장 작은 자연수를 적용한 명제가 참임을 보이고, 임의의 자연수 k를 적용한 명제가 참임을 가정했을 때, k+1을 적용한 명제가 항상 참임을 보이면 해당 명제는 모든 자연수에 대해 참임이 증명된다. 즉, *P(n)*이 모든 자연수 n에 대해서 참임을 보이기 위해서 다음 두 명제가 참임을 증명하면 된다.
```
P(1)이 성립
P(k)가 성립한다고 가정했을 때, P(k+1)이 항상 성립
```
<br><br>

## **🔨 재귀와 수학적 귀납법의 관계**

 그렇다면 재귀와 수학적 귀납법은 어떠한 관계를 맺고 있을까? 재귀는 수학적 귀납법의 ***원리***를 이용한다. 주어진 자연수 n에 대해 P(n)의 값을 구하는 문제를 생각해보자. 재귀를 이용하는 알고리즘에서는 대부분 P(n)을 구하기 위해서 P(1), P(2), P(3), ... , P(n-1)을 차례대로 구하는 방식으로 문제를 해결한다. 그리고 P(k)의 값이 P(k+1)을 구하는데 이용된다. (k는 자연수) 이러한 과정에서 재귀함수는 P(n)이라는 문제를 P(1)이라는 문제까지 ***쪼개는*** 역할을 한다. (이 때, 재귀함수는 P(n)의 값이 존재한다고 가정함) 이렇게 문제를 P(1)까지 쪼개는(?) 과정을 마치고, P(1)의 값을 마지막으로 호출된 함수의 인스턴스에서 반환하면, 지금까지 호출된 함수의 인스턴스들이 각각 값을 차례대로 반환하면서 P(2), P(3), ... , P(n-1), P(n)의 값들을 구할 수 있게된다. <br><br>    

## **🔎 재귀함수의 구성 및 특징**

 다음 코드는 [B.O.J. 1074번](https://www.acmicpc.net/problem/1074)을 해결하기 위해 구현한 재귀함수이다. 이 함수를 통해 재귀함수의 구성과 이로 인한 특징을 알아보자.
 
```cpp
int func(int r, int c, int N){
  if(N == 1){

    return r * 2 + c;
  }

  if(r < pow(2, N - 1)){
    if(c < pow(2, N - 1)){

      return func(r, c, N - 1);
    }
    else{

      return pow(2, 2 * N - 2) + func(r, c - pow(2, N - 1), N - 1);
    }
  }
  else{
    if(c < pow(2, N - 1)){

      return pow(2, 2 * N - 2) * 2 + func(r - pow(2, N - 1), c, N - 1);
    }
    else{

      return pow(2, 2 * N - 2) * 3 + func(r - pow(2, N - 1), c - pow(2, N - 1), N - 1);
    }    
  }
}
```

### *Base Case* ###
---

```cpp
if(N == 1){

  return r * 2 + c;
}
```

 주어진 함수 func에서 위 코드는 Base Case를 처리하는 부분이다. Base Case는 재귀함수가 더 이상 쪼갤 수 없는, 또는 쪼갤 필요가 없는 문제를 말한다. 위에서 언급한 P(n) 문제에서는 P(1)이 바로 Base Case에 해당한다. 재귀와 수학적 귀납법에서 언급했듯이 Base Case의 답이 반환됨으로써 수학적 귀납법의 원리에 의해 문제의 답이 차례대로 구해지기 때문에, 재귀함수를 정의할 때 Base Case를 처리하는 부분은 반드시 존재해야 한다. 또한 재귀함수는 반드시 Base Case에 수렴하는 방향으로 정의되어야 한다.<br><br>

### *무책임한(?) 파트* ###
--- 

```cpp
if(r < pow(2, N - 1)){
  if(c < pow(2, N - 1)){

    return func(r, c, N - 1);
  }
  else{

    return pow(2, 2 * N - 2) + func(r, c - pow(2, N - 1), N - 1);
  }
}
else{
  if(c < pow(2, N - 1)){

    return pow(2, 2 * N - 2) * 2 + func(r - pow(2, N - 1), c, N - 1);
  }
  else{

    return pow(2, 2 * N - 2) * 3 + func(r - pow(2, N - 1), c - pow(2, N - 1), N - 1);
  }    
}
```

 주어진 함수 func에서 위 코드는 문제를 쪼개는 역할을 수행하는 부분이다. 이 파트를 '무책임한 파트'라고 이름붙인 이유도 바로 이 부분의 역할 때문이다. 문제를 쪼개는 것은 곧 자신과 같은 함수의 또 다른 인스턴스에게 더욱 작아진 입력값에 대한 문제를 해결해줄 것을 요청하는 것이다. 따라서 이와 같은 재귀함수를 짤 때는 현재 입력값에 대한 문제 해결 방법만을 생각하면 된다. 위에 제시된 func이라는 재귀함수를 예시로 들어보자. func의 입력값은 행과 열에 대한 정보(int r, int c)와 현재 배열의 크기에 대한 정보(int N)이다. 이 때, 입력값이 Base Case에 해당하는 값이 아니라면, 배열을 4등분하여 r행 c열의 원소가 어느 부분에 있는지 파악한다. 이 부분은 if-else문으로 처리한다. 4등분한 배열의 각 부분은 차례대로 방문되므로 각 부분의 순서에 맞게 계산한 수를 더해주고, 이제 4분의 1 사이즈가 되어버린 배열은 이 함수에 의해 쪼개진 또 다른 문제이다. 따라서 func에 더욱 작아진 입력값을 입력해 호출하여 답을 구한 뒤 더해주어 그 값을 반환하면, 배열 크기에 대한 입력값이 N일 때의 답이 구해지는 것이다. 이 함수를 보면 알 수 있듯이, 함수 내에서 크기가 2^2(N-1)인 배열 내에서 원소의 순서를 구하는 부분은 같은 함수의 또 다른 인스턴스를 호출하고 있다.

```cpp
return func(r, c, N - 1);
return pow(2, 2 * N - 2) + func(r, c - pow(2, N - 1), N - 1);
return pow(2, 2 * N - 2) * 2 + func(r - pow(2, N - 1), c, N - 1);
return pow(2, 2 * N - 2) * 3 + func(r - pow(2, N - 1), c - pow(2, N - 1), N - 1);
```

 이와 같이 재귀함수를 정의할 때, 문제를 쪼개는 부분을 잘 정의해야 한다. 여기서 '잘' 정의한다는 의미는 첫째로 작아진 입력값을 같은 함수의 또 다른 인스턴스에게 '무책임하게' 전달해야 한다는 뜻이며, 둘째로 이러한 과정을 거쳐 점점 작아진 문제는 Base Case에 수렴해야 한다는 뜻이다.
